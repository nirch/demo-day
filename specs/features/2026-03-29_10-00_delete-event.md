## Feature: Delete Event

### Problem Statement
Admins have no way to remove events they no longer need. Draft events (created by mistake or abandoned) clutter the event list with no cleanup path.

### User Stories
- As an **admin**, I want to delete a draft event from the event details page so that I can remove events that are no longer needed.
- As an **admin**, I want to be warned before deletion so that I don't accidentally destroy an event and all its related data.
- As an **admin** viewing a non-draft event, I should **not** see a delete option, so I don't accidentally try to delete an event with submitted scores.

### Acceptance Criteria
- [ ] A "Delete Event" button appears at the bottom of the Event Detail page, **only** when the event status is `draft`
- [ ] The button is visible to any authenticated admin, not just the creator
- [ ] Clicking the button opens a confirmation modal: *"Are you sure you want to delete this event? This will permanently remove all teams, criteria, scores, and judge assignments."*
- [ ] Confirming the modal sends a `DELETE /api/events/:id` request
- [ ] The backend verifies the user is an admin and the event is in `draft` status before deleting
- [ ] On success, the user is redirected to the event list page
- [ ] On failure (non-draft, not found, unauthorized), a descriptive error is shown
- [ ] The delete button does **not** appear on the event list page
- [ ] The database cascades the delete to all related records (teams, criteria, event_judges, scores, team_comments) — already configured via migrations

### UI/UX Notes

**Delete Button (bottom of Event Detail Page):**
- Placed below the last section (Scoring Criteria), separated by spacing
- Destructive variant styling (red/danger color from design tokens)
- Text: "Delete Event"
- States: default, hover, loading (while request is in flight), hidden when event is not in draft status

**Confirmation Modal:**
- Title: "Delete Event"
- Body: "Are you sure you want to delete this event? This will permanently remove all teams, criteria, scores, and judge assignments."
- Two buttons: "Cancel" (secondary) and "Delete" (destructive)
- Cancel closes the modal, Delete fires the API call
- While deleting: Delete button shows loading state, Cancel is disabled

**States:**
- **Empty/Loading/Error on the page itself:** N/A — leverages existing Event Detail page states
- **API error after confirm:** Toast or inline error in the modal — "Failed to delete event. Please try again."
- **Success:** Redirect to event list page

**Responsive:** Modal is centered on desktop, bottom-sheet style or centered on mobile (follow existing modal patterns in the app).

### API Design

```
DELETE /api/events/:id
Auth: JWT (admin role required)
Validations: event must exist, event.status must be 'draft'
Success: 200 { data: { message: "Event deleted successfully" }, error: null }
Errors:
  401 — unauthorized
  403 — not an admin
  404 — event not found
  409 — event is not in draft status
```

### Out of Scope
- Soft delete / archiving events
- Undo after deletion
- Deleting events in non-draft status
- Delete from the event list page
- Batch delete of multiple events
